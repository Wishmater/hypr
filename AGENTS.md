# Hyprland Lua Config

## LSP Support

`hl.meta.lua` provides LuaLS type stubs for the Hyprland `hl.*` API
(completions, type checking, diagnostics). It is autogenerated from the
Hyprland C++ source.

### Regenerate after Hyprland updates

```bash
# Clone Hyprland (shallow, ~15s)
git clone --depth 1 https://github.com/hyprwm/Hyprland.git /tmp/hypr-stubs

# Generate stubs (needs python3)
nix-shell -p python3 --run \
  "python3 /tmp/hypr-stubs/meta/generateLuaStubs.py \
    --root /tmp/hypr-stubs \
    --output $HOME/.config/hypr/hl.meta.lua"

# Clean up
rm -rf /tmp/hypr-stubs
```

If you're not on NixOS, the last line simplifies to:
```bash
python3 meta/generateLuaStubs.py --output ~/.config/hypr/hl.meta.lua
```
(run from inside the cloned repo)

## File overview

| File                  | Purpose                              |
|-----------------------|--------------------------------------|
| `hyprland.lua`        | Entry point, requires all modules    |
| `vars.lua`            | Shared variables (apps, monitors)    |
| `style.lua`           | General, decoration, animations      |
| `input.lua`           | Keyboard, mouse, cursor              |
| `keybinds.lua`        | All key bindings                     |
| `window_rules.lua`    | Window rules, popup/game tags        |
| `layer_rules.lua`     | Layer shell rules                    |
| `workspaces.lua`      | Workspace assignments, layout opts   |
| `monitors.lua`        | Monitor config, render quirks        |
| `layout.lua`          | Master/dwindle/scrolling layout opts |
| `env.lua`             | Environment variables                |
| `autostart.lua`       | exec-once on hyprland.start          |
| `permissions.lua`     | Security permissions                 |
| `hl.meta.lua`         | LuaLS type stubs (autogenerated)     |
| `.luarc.json`         | LuaLS workspace config               |
